A Claude Code plugin

/ship

Commit and push without burning your context. The whole flow runs on a Haiku subagent — your main session never sees the diff.

$ /plugin marketplace add geberele/claude-plugins
Claude Code · slash command runs on Haiku MIT
Why it exists

The cost of shipping ceremony

The problem

Every "commit and push" turn looks the same: read git status, read the diff, scan recent commits to match the convention, draft a message, commit, push, handle the missing upstream. It works — but it dumps the full diff and twenty log entries into your main session every time. Your context fills up with shipping ceremony, and you pay for those tokens at your main model's rate.

The fix

/ship hands the whole flow to a dedicated subagent (git-shipper) running on Haiku. It has its own context window and exactly one job. Your main session sees a short summary — commit hash, message, push result — and nothing else. The diff and the log get thrown away.

What you get

Four things, delivered every time

// 01

Lean main context

No diffs, no git log dumps polluting the conversation you're actually having.

// 02

Lower cost

The expensive part — reading the diff, drafting the message — runs on Haiku, not your main model.

// 03

Repo-aware messages

Detects Conventional Commits, ticket-prefixed (PROJ-1234:), or free-form, and matches the dominant style.

// 04

Safe by default

Never --force, never --no-verify, never touches your source code.

How it works

One command. One subagent. One summary back.

/ship is a thin slash command that delegates to git-shipper, a Bash-only subagent pinned to Haiku. Everything happens in the subagent's context — the only thing that comes back to your main session is the result. 

you ▸ /ship
 └─ main session
      └─ delegates to git-shipper  (Haiku · Bash only)
           ├─ git status / git diff       # read changes
           ├─ git log --oneline -20       # detect convention
           ├─ git branch --show-current   # extract ticket ID
           ├─ git add / commit / push     # ship
           └─ returns: hash + message + push result
 └─ main session sees only the summary

What it will and won't do

Will

  • Stage your changes, write the message, commit, push.
  • Set the upstream on first push if missing.
  • Stop and tell you if the diff is empty.

Won't

  • git push --force, even if asked.
  • Skip hooks (--no-verify) or bypass signing.
  • Touch source code, run tests, or create branches.
Install & use

Two lines to install. One word to ship.

Requires Claude Code.

$/plugin marketplace add geberele/claude-plugins
$/plugin install ship@geberele

Usage

$/ship # infer everything from the diff
$/ship fixes flaky auth test # add extra context for the message
